	3.	Make Elo dynamic and smarter
	•	Adjust Elo based on margin of victory, not just win/loss.
	•	Give more weight to recent games versus old games.
	•	Adjust for home-field advantage dynamically.
	•	Optionally tweak Elo for injuries or high-variance teams.
	4.	Generate better features for XGBoost
	•	Include team stats like offensive/defensive efficiency, turnovers, red-zone performance, etc.
	•	Include situational features: days of rest, travel distance, weather (for NFL), short weeks.
	•	Include rolling averages over last 3–5 games, not just season stats.
	•	Include the current Elo ratings for both teams.
	•	If you want, include betting lines or spreads as a feature for market context.
	5.	Train XGBoost and calibrate it
	•	Use your generated features to train the model.
	•	Tune hyperparameters (max depth, learning rate, number of trees, etc.) for best performance.
	•	Optionally calibrate the predicted probabilities so they better reflect actual win chances.
	6.	Build an ensemble
	•	Combine Elo predictions, XGBoost predictions, and optionally a simple logistic regression or Bayesian model into a final prediction.
	•	Use a simple stacking model where each individual model’s output is fed into a meta-model to get the final pick.
	7.	Set up automatic updates
	•	Replit should pull stats automatically from your API if available.
	•	schedules.py should be a fallback so you can manually add games for the season if the API fails.
	•	Each week, update Elo and retrain XGBoost with new games to keep predictions current.
	8.	Output predictions in the GUI
	•	Predictions should show up for every game from the beginning of the season.
	•	Include confidence levels (probabilities) for each pick.
	•	Highlight favorites vs underdogs and allow filtering by team, week, or confidence.
